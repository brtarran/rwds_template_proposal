Adding non-text elements

You can add code like this:

set_signif_par()
plot(1:4, 1:4, col=1:4, pch=19, cex=3, xlab="", ylab="",
     main = "My Significance Plot",
     sub = "Source: data source")
abline(h=1:4, v=1:4, col = "lightgrey")

Not only in R, but also in Python:

import pandas as pd
print("Hello this is Python code")

or Julia:

using Plots

plot(sin, 
     x->sin(2x), 
     0, 
     2π, 
     leg=false, 
     fill=(0,:lavender))

If you want to show how to write a specific code-chunk in Quarto, you can use the echo: fenced option to show the code in the output.

```{r}
#| code-line-numbers: true
# This is a fenced code block with some R code, that show the option code-line-numbers: true.
# Note that echo: fenced is hidden in the output.
x <- 1:10
y <- x^2
plot(x, y)
```

You can add a footnote to your writing1

By adding your references as bibtex to the references.bib file in this repository, you not only maintain your references automatically in the right format, but you can also cite your references directly as recommended by Doe (2020), or indirectly (compare to Doe 2020).

If your tutorial includes tips on special keyboard shortcuts, you can typeset them like this: To print, press Shift-Ctrl-P. To open an existing new project, press .

You can write equations in-line like this \(y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \epsilon \text{, }\) or in a separate block like this \[ y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \epsilon \text{. } \tag{1}\]

Rather than creating diagrams in Canva, Figma, or PowerPoint and pasting them into your article as an image, consider using Mermaid. Note how the chunk options fore Mermaid start with %%| and that you need to set the option eval: true and echo: false to only generate your graph and not the code that created it.
